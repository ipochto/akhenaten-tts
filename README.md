
```
Usage:
  akhenaten-tts [OPTION...]

  -h, --help        Print usage
  -f, --figure arg  Персонаж, для которого озвучиваем фразу. Должен быть прописан в конфигурационно скрипте (--config)
  -p, --phrase arg  Текст, озвучиваемой, фразы ("Фраза должна быть указана в кавычках")
  -l, --lang arg    Двубуквенный код языка - en, ru и т.д.
  -o, --output arg  Имя выходного аудиио файла. На настоящий момент пока только WAV генерирует.
  -c, --config arg  Конфигурационный скрипт, содержащий описание голосовых моделей и перснонажей (default: tts-config.lua)
```

По мере необходимости файлы голосовых моделей ищщутся в кэше, если там нет, то скачиваются автоматически.

Вообще можно поиграться с разными голосами. 
Предварительно повыбирать [здесь](https://rhasspy.github.io/piper-samples/), затем прописываем их в [tts-config.lua](src/scripts/tts-config.lua).

```lua
local voices = {
  en = { -- язык голосовой модели
    ["libritts"] = { -- название голосовой модели модели
      url = "https://huggingface.co/rhasspy/piper-voices/resolve/main/en/en_US/libritts/high/en_US-libritts-high.onnx", -- адрес, откуда скачиваем
      filename = "en/en_US/libritts/high/en_US-libritts-high.onnx", -- куда кладём скачанную модель, и где её потому будем искать (путь отностительный)
      speakers = { -- каждая модель может содержать несколько спикеров
        num = 904, -- количество спикеров в модели
        male = { 2, 3, 5, 6, 7, 8, 16, 19, 20, 22, 24, 27, 28, 31, 32, 34, 35}, -- справочно
        female = { 0, 1, 4, 9, 10, 11, 12, 13, 14, 15, 17, 18, 21, 23, 25, 26, 29, 30, 33 } -- справочно
      }
    },
    ...
  }
  ...
}
...
local figures = {
  ["immigrant"] = {                       -- имя персонажа
    en = {                                -- язык
      voiceModel = voices.en["libritts"], -- модель должна быть определена в voices (см. выше)
      speaker = 3 },                      -- спикер
    ru = {}
  },
  ...
}
```
